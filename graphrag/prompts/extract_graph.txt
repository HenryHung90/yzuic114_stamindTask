# JavaScript Objects

## What are Objects?
An **object** is a variable that can hold multiple key-value pairs. Objects are collections of properties (keys) and their associated values. They can describe anything, such as houses, cars, people, animals, or other subjects.

- Different objects can have the same properties, but their property values may differ.
- Objects can also have methods that perform actions, which may be executed at different times.

## Creating a JavaScript Object

### Using Object Literals
An **object literal** is a concise way to create an object. It consists of key-value pairs enclosed in curly braces `{{}}`.

```javascript
const car = {{
  brand: "Fiat",
  model: "500",
  color: "white"
}};
```

**Note:** It is recommended to declare objects using the `const` keyword. While you cannot reassign the `const` variable to a different object, you can still modify its properties and values.

### Using the `new Object` Keyword
You can also create objects using the `new Object()` syntax:

```javascript
const car = new Object();
car.brand = "Fiat";
car.model = "500";
car.color = "white";
```

**Note:** While both methods achieve the same result, using object literals is simpler, faster, and more readable.

## Accessing Object Properties
Object properties can be accessed in two ways:
- **Dot notation**: `object.property`
- **Bracket notation**: `object["property"]`

```javascript
console.log(car.brand); // Output: Fiat
console.log(car["model"]); // Output: 500
```

## JavaScript Object Methods
Object methods are actions (functions) that can be performed on objects. Methods are stored as property values.

```javascript
const person = {{
  firstName: "John",
  lastName: "Doe",
  fullName: function() {{
    return this.firstName + " " + this.lastName;
  }}
}};

console.log(person.fullName()); // Output: John Doe
```

**Note:** In methods, `this` refers to the object itself. For example, `this.firstName` refers to the `firstName` property of the `person` object.

## Displaying JavaScript Objects

### Displaying Objects
Displaying an object directly will output `[object Object]`.

```javascript
console.log(person); // Output: [object Object]
```

### Displaying Object Properties
To display specific properties, concatenate them into a string:

```javascript
console.log("Name: " + person.firstName + " " + person.lastName);
```

## Object Constructor Functions
Sometimes, you need to create multiple objects of the same type. You can use **constructor functions** to define a reusable object type.

```javascript
function Person(firstName, lastName) {{
  this.firstName = firstName;
  this.lastName = lastName;
}}

const person1 = new Person("John", "Doe");
const person2 = new Person("Jane", "Smith");

console.log(person1.firstName); // Output: John
console.log(person2.lastName); // Output: Smith
```

**Note:** Constructor functions are named with an uppercase first letter by convention.

## Summary
- **Objects** are containers for properties and methods.
- **Properties** are named values (e.g., strings, numbers, functions, or other objects).
- **Methods** are functions stored as properties.
- **Constructor functions** create object prototypes.

## JavaScript and Objects
In JavaScript, almost everything is an object. All values except primitives are objects.

### JavaScript Primitives
Primitive data types can only store a single value. JavaScript defines 7 primitive data types:
- `string`
- `number`
- `boolean`
- `bigint`
- `null`
- `undefined`
- `symbol`

# HTML Attributes

HTML attributes provide additional information about HTML elements.

## The `href` Attribute
The `<a>` tag defines a hyperlink. The `href` attribute specifies the URL of the page the link goes to.

#### 程式碼範例
```html
<a href="https://www.example.com">Visit Example</a>
```

#### Example
You will learn more about links in our **HTML Links** chapter.

## The `src` Attribute
The `<img>` tag is used to embed an image in an HTML page. The `src` attribute specifies the path to the image to be displayed.

#### 程式碼範例
```html
<img src="img_girl.jpg" alt="Girl">
```

### Ways to Specify the URL in `src`
1. **Absolute URL**: Links to an external image hosted on another website.  
   Example:  
   ```html
   <img src="https://www.example.com/images/img_girl.jpg" alt="Girl">
   ```
   **Notes**: External images might be under copyright. You cannot control external images; they can suddenly be removed or changed.

2. **Relative URL**: Links to an image hosted within the website.  
   - If the URL begins without a slash, it will be relative to the current page.  
     Example:  
     ```html
     <img src="img_girl.jpg" alt="Girl">
     ```
   - If the URL begins with a slash, it will be relative to the domain.  
     Example:  
     ```html
     <img src="/images/img_girl.jpg" alt="Girl">
     ```

   **Tip**: It is almost always best to use relative URLs. They will not break if you change the domain.

## The `width` and `height` Attributes
The `<img>` tag should also contain the `width` and `height` attributes, which specify the width and height of the image in pixels.

#### 程式碼範例
```html
<img src="img_girl.jpg" width="500" height="600" alt="Girl">
```

## The `alt` Attribute
The required `alt` attribute for the `<img>` tag specifies alternate text for an image if the image cannot be displayed.

#### 程式碼範例
```html
<img src="img_does_not_exist.jpg" alt="Image not found">
```

#### Example
See what happens if we try to display an image that does not exist.  
You will learn more about images in our **HTML Images** chapter.

## The `style` Attribute
The `style` attribute is used to add styles to an element, such as color, font size, and more.

#### 程式碼範例
```html
<p style="color: red; font-size: 20px;">This is styled text.</p>
```

#### Example
You will learn more about styles in our **HTML Styles** chapter.

## The `lang` Attribute
You should always include the `lang` attribute inside the `<html>` tag to declare the language of the webpage. This assists search engines and browsers.

#### Example
```html
<html lang="en">
```
Country codes can also be added to the language code in the `lang` attribute.  
Example:  
```html
<html lang="en-US">
```
You can see all the language codes in our **HTML Language Code Reference**.

## The `title` Attribute
The `title` attribute defines extra information about an element. The value of the `title` attribute will be displayed as a tooltip when you mouse over the element.

#### 程式碼範例
```html
<p title="This is a tooltip">Hover over this text.</p>
```

## Best Practices for HTML Attributes
- **Always Use Lowercase Attributes**  
  The HTML standard does not require lowercase attribute names, but W3C recommends lowercase attributes in HTML and demands lowercase attributes for stricter document types like XHTML.  
  Example:  
  ```html
  <p title="example"></p>
  ```

- **Always Quote Attribute Values**  
  The HTML standard does not require quotes around attribute values, but W3C recommends quotes in HTML and demands quotes for stricter document types like XHTML.  
  Example:  
  ```html
  <p title="example"></p>
  ```

  **Single or Double Quotes**  
  Double quotes are the most common, but single quotes can be used when the attribute value contains double quotes.  
  Example:  
  ```html
  <p title='This "tooltip" uses single quotes'></p>
  ```
